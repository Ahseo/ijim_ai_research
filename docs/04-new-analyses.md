# 필요한 신규 통계 분석

## 우선순위 1: Event-Study / 동적 DID (반드시 먼저 실행)

### 수식
```
Y_{it} = α_i + γ_t + Σ_{k≠-1} β_k (Treat_i × 1[t=k]) + X_{it}'δ + ε_{it}
```
- `k` = AI 도입 대비 상대 주차 (-4 ~ +4)
- `k = -1` 기준기간으로 생략
- `β_k` (k < 0) = 사전 추세 계수 (≈ 0이어야 함)
- `β_k` (k ≥ 0) = 동적 처리 효과

### 산출물
- 그림: 95% 신뢰구간 포함 계수 플롯
- 사전 처리 계수에 대한 결합 F-검정 (p > 0.10 필요)
- **GO/NO-GO 관문**: 사전 추세 실패 시 전체 식별 전략 재검토

### 대응: R1-3A, C2

---

## 우선순위 2: Oster (2019) Bounds

### 단계
1. 통제변수 없이 추정 → `β_비통제`, `R²_비통제`
2. 전체 통제변수로 추정 → `β_통제`, `R²_통제`
3. `R_max = min(1, 1.3 × R²_통제)` 설정
4. δ* 계산

### 해석
- δ* > 1 → 비관측 변수가 모든 관측 변수를 합친 것보다 더 커야 효과를 설명 가능
- 보고: "비관측 선택이 관측 공변량 전체의 [X]배여야 추정 효과를 무효화할 수 있다"

### 도구
- Stata: `psacalc` 또는 `regsensitivity`
- R: `sensemakr`

### 대응: R1-3C, R2-7

---

## 우선순위 3: 처리 강도 / 용량-반응

### 변수 구성
1. **AI 사용 비율**: `AIShare_i = (AI 배정 주문 수) / (사후 기간 전체 주문 수)`
2. **AI 활성 일수 비율**: `AIDayShare_i = (AI 활성 일수) / (전체 활동 일수)`

### 수식

**연속형:**
```
Y_{it} = α_i + γ_t + β₁(AIShare_i × Post_t) + X_{it}'δ + ε_{it}
```

**사분위 구간:**
```
Y_{it} = α_i + γ_t + Σ_{q=1}^{4} β_q × 1[AIShare_i ∈ Q_q] × Post_t + X_{it}'δ + ε_{it}
```

### 산출물
- 표: AI 사용 사분위별 처리 효과
- 그림: 신뢰 구간 포함 용량-반응 곡선
- 검정: 단조성 (Q4 효과 > Q1 효과, p-값)

### 핵심 논거
단조적 용량-반응은 순수 선택으로 설명하기 어려움 — 비관측 능력이 사용 강도에 정확히 비례해야 한다는 훨씬 강한 가정 필요

### 주의: 용량 자체가 내생적. "메커니즘 증거"로 프레이밍하고, 독립적 인과 증명으로는 아님

### 대응: R1-3B

---

## 우선순위 4: 안정 근로자 하위표본

### "안정"의 정의
도입 전후 변화가 개인 사전 평균의 ±1 SD 이내인 라이더:
- 주간 근무일수
- 일일 근무시간
- 서비스 지역
- 주문량

### 실행
주 DID를 제한된 표본에서 재추정. 표본 크기 감소 보고. 효과가 전체 표본 95% CI 내에 있어야 함.

### 민감도: ±0.5 SD, ±1.5 SD 기준으로도 보고

### 대응: R1-3C, R2-7

---

## 우선순위 5: Gini / Theil 불평등 분해

### 지표
- 일일 생산성 분포의 Gini 계수 (도입 전 vs 후)
- Theil 지수 분해 (집단 간 vs 집단 내)
- P90/P10, P75/P25 생산성 비율

### 비교
- 채택자 vs 비채택자, 도입 전 vs 후
- 숙련 수준(proficiency) 그룹별

### 대응: R1-M2

---

## 우선순위 6: 근로자-고객 매개 분석

### 데이터가 라이더-고객 연결 가능 시:
2단계 매개:
1. 1단계: AI → 근로자 효율성 (주문/시간, 유휴 시간)
2. 2단계: 근로자 효율성 → 고객 대기 시간

### 불가능 시:
집계 분석 — 근로자 개선이 큰 지점/기간에서 고객 개선도 큰지 확인

### 대응: AE-2

---

## 우선순위 7: 표본 대표성

### 실행
- 표본 인구통계를 플랫폼 전체 통계와 비교 (파트너로부터 입수 가능 시)
- 검정력 분석: N=336 매칭 쌍, 효과 크기 ~3%로 검정력 산출
- 80% 검정력에서 최소 검출 가능 효과 크기 보고

### 대응: R2-6

---

## 우선순위 8: 기간 내 학습 역학

### 사후 기간 내 일별 데이터 존재 시:
AI 효과가 1→4주차에 걸쳐 강화되는지 검정
```
Y_{it} = α_i + γ_t + Σ_{w=1}^{4} β_w(Treat_i × Week_w) + X'δ + ε_{it}
```
- β_w 증가 → 학습 효과, 중기 적응 지지
- β_w 평탄 → 즉각 효과, 학습 없음

### 대응: R2-9

---

## 우선순위 9: 대안 처리 정의

| 정의 | 기준 |
|-----|------|
| 집중 채택자 | 사후 기간 주문의 50% 초과가 AI 배정 |
| 지속 채택자 | 사후 기간 4주 중 3주 이상 AI 사용 |
| 전반기 채택자 | 1-2주차에만 채택한 경우 |
| 연속형 | AIShare_i 직접 사용 (이진 분류 없음) |

각 정의로 주요 효과 재추정. 일관성은 인과 주장을 강화.

### 대응: R1-3B

---

## 우선순위 10: SUTVA (파급 효과) 검정

### 우려
처리군 라이더가 AI로 좋은 주문을 받으면 → 비처리군은 나쁜 주문 → DID 추정치 부풀려짐

### 검정
비채택자 성과가 사후 기간에 하락하는지 확인 (부정적 파급 시사)

### 대응: 일반 강건성
